# Qube 블록 회전 — 한 손 터치 제스처 방안 정리

**작성일**: 2026-03-18
**상태**: 구현 완료 — 하단 스와이프 방식 채택

### 구현된 방식

프리뷰 블록 회전은 **드래그 전 하단 스와이프** 방식으로 구현됨:
- 화면 하단 40%에서 좌우 스와이프 → 첫 번째 블록 ±90° 회전
- 스와이프 임계값 80px, 수평 > 수직×1.5 조건
- 연속 스와이프 시 80px마다 추가 회전
- EaseOutBack 회전 애니메이션 (0.2초, 블록 셀만 회전)
- 빠른 연속 회전 시 애니메이션 없이 즉시 스냅
- 틱 사운드 재생 (프로시저럴, 회전당 1회)

아래는 검토했던 방안들의 기록:

---

## 1. 배경

- 블록 회전은 Phase 0에서 제거됨 (현재는 큐 생성 시 랜덤 회전만 적용)
- `QubeBlock.ApplyRotation()` static 메서드는 잔존
- 목표: 한 손 터치로 블록 이동 + 회전 + 배치를 모두 처리

---

## 2. 방안 비교표

| | A. 더블탭 | B. 플릭 | C. 홀드-회전 | D. 원형 제스처 |
|---|---|---|---|---|
| **제스처** | 드래그 중 두 번째 손가락 탭 | 드래그 중 빠른 위/아래 플릭 | 0.3초 정지 → 좌/우 스와이프 | 드래그 중 원 그리기 |
| **한 손가락** | X (두 손가락) | O | O | O |
| **직관성** | 높음 | 중간 | 중간 | 매우 높음 |
| **오조작 위험** | 낮음 | 높음 | 낮음 | 중간 |
| **속도** | 빠름 | 매우 빠름 | 느림 (홀드 대기) | 중간 |
| **구현 난이도** | 쉬움 | 중간 (threshold 튜닝) | 중간 | 높음 (각도 누적 알고리즘) |
| **위치 영향** | 없음 | 있음 (플릭 방향 이동) | 없음 (모드 분리) | 있음 → freeze로 해결 |

## 3. 각 방안 상세

### A. 더블탭 회전

```
[엄지로 드래그 이동] + [검지로 화면 아무 곳 탭] → 90° 회전
```

- 유일한 2손가락 방식, 하지만 가장 단순하고 안정적
- 이동과 회전이 완전히 독립 → 오조작 없음

### B. 스와이프 플릭 회전

```
[드래그 중 빠르게 위로 플릭] → 시계방향 90°
[드래그 중 빠르게 아래로 플릭] → 반시계방향 90°
```

- velocity 임계값으로 이동/회전 구분
- 가장 빠르지만 의도치 않은 회전 위험 → 튜닝 필수

### C. 홀드-회전 모드

```
[드래그 → 0.3초 정지] → 강한 햅틱 → 회전 모드 진입
[좌/우 스와이프] → 회전
[다시 이동] → 이동 모드 복귀
```

- 이동/회전 모드를 명확히 분리 → 가장 정확
- 홀드 대기 시간이 플레이 템포를 늦출 수 있음

### D. 원형 제스처 회전

```
[드래그 중 시계방향 원 그리기] → 시계방향 90°
[드래그 중 반시계방향 원 그리기] → 반시계방향 90°
```

- "돌린다 → 회전한다" — 물리적으로 가장 직관적
- 원 감지 중 위치 freeze 필요
- 각도 누적 270°~330° 도달 시 발동 (완전한 원 불필요)

#### D. 원형 감지 알고리즘

1. 드래그 중 최근 20~30프레임의 **무게중심(centroid)** 계산
2. 매 프레임 centroid 기준 현재 터치의 **각도 변화량(delta angle)** 누적
3. 누적 각도가 ±270°~330° 도달 시 회전 발동 + 누적 리셋
4. 직선 이동(centroid가 계속 이동) 시에는 각도가 누적되지 않음

| 파라미터 | 값 | 설명 |
|----------|-----|------|
| `ROTATION_THRESHOLD` | 270°~330° | 완전한 원이 아니어도 발동 |
| `MIN_RADIUS` | 30px | 너무 작은 원은 무시 (떨림 방지) |
| `MAX_RADIUS` | 150px | 너무 큰 원은 이동으로 처리 |
| `SAMPLE_COUNT` | 최근 20~30프레임 | centroid 계산용 샘플 |
| `RESET_ON_LINEAR` | true | 직선 이동 감지 시 누적 리셋 |

## 4. 조합 추천

단독 사용보다 **조합**이 더 실용적:

| 조합 | 구성 | 적합한 유저 |
|------|------|------------|
| **B + D** | 빠른 플릭(숙련자) + 원형(초보자) | 모든 유저 — 자연스럽게 취향대로 사용 |
| **C + A** | 홀드-회전(기본) + 더블탭(보조) | 정밀 배치 중시 유저 |
| **D 단독** | 원형 제스처만 | 최소 UI, 직관적 경험 우선 |

## 5. 공통 구현 요소

### 5.1 QubeBlock 회전 메서드 복원

- `Rotate(int direction)` — `currentCells`에 `ApplyRotation` 적용
- 비주얼 + 고스트 재생성
- Wall Kick: 회전 후 그리드 밖이면 1~2칸 밀어서 재시도

### 5.2 햅틱 피드백 (모든 방안 공통)

| 이벤트 | 강도 | 패턴 |
|--------|------|------|
| 그리드 셀 스냅 | Light | 짧게 1회 |
| 회전 진행 힌트 (D의 90° 단위) | Light | tick |
| **회전 완료** | Medium | 딸깍 1회 |
| 회전 불가 | Heavy | 빠르게 2회 |
| 배치 성공 | Medium | 1회 |
| 배치 취소 | Heavy | 길게 1회 |

Unity 구현: `Handheld.Vibrate()` 또는 iOS `UIImpactFeedbackGenerator` / Android `VibrationEffect` 네이티브 플러그인

### 5.3 InputHandler 확장

- 기존 `UpdateDragging()`에 제스처 감지 로직 추가
- 회전 감지 시 `InputAction.Rotate` 반환
