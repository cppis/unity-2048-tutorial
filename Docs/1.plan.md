## 2. 아키텍처 현황

### 클래스 구조

```
QubeGameManager (싱글톤, DefaultExecutionOrder -1)
├── 프리뷰 슬롯 드래그 이벤트 구독 (OnPreviewSlotDragStarted)
├── SpawnBlockFromQueue(slotIndex) — 큐에서 블록 생성
├── CancelCurrentBlock() — 드래그 취소 시 블록 제거
├── CanAnyQueuedBlockPlace() — 게임오버 판정 (큐 전체 검사)
├── 난이도 시스템 (QubeDifficulty ScriptableObject)
├── 점수/UI 업데이트
└── 게임 상태 (isGameOver, isPlacingBlock)

QubeInputHandler (입력 통합 — 신규)
├── 키보드: WASD 이동 + Space 배치
├── 터치: 그리드 위 드래그 + 릴리즈 배치
├── 프리뷰 드래그: StartDragFromPreview() → 그리드 스냅 이동
├── 드래그 취소: 그리드 밖 릴리즈 → CancelDrag 액션
└── ScreenToGridPosition() — 화면 좌표 → 그리드 좌표 변환

QubeGrid (가변 크기 그리드 컨테이너)
├── static WIDTH/HEIGHT (SetSize()로 변경 가능)
├── cells[WIDTH, HEIGHT] 배열 (동적 할당)
├── GridLayoutGroup (80x80, spacing 5)
├── PlacedBlocks 컨테이너 (배치된 셀 + 채움 조각)
├── RebuildGrid() — 난이도 변경 시 그리드 재생성
└── RemoveFillPieces() — 소거 시 채움 조각도 함께 제거

QubeBlock (드래그 조작 블록)
├── QubeBlockShape (ScriptableObject)
├── 이동: Move() + SetPosition() (드래그용 직접 위치 설정)
├── Ghost 미리보기 (배치 가능: 반투명 블록색, 불가: 반투명 빨간색)
├── 이동 중 셀 크기 = cellSize + spacing (구분선 없이 표시)
├── 배치 시 셀 크기 = cellSize (그리드 간격으로 블록 구분)
├── Place() — 같은 블록 인접 셀 사이에 채움 조각(Fill) 생성
├── Initialize() 오버로드 — 사전 회전된 셀 배열 수신
└── ApplyRotation() static — 큐에서 사전 회전 시 사용

QubeBlockQueue (다음 블록 큐)
├── Queue<QubeBlockEntry> (5개, Shape + 사전 회전 셀)
├── Initialize / Dequeue / GetPreview
├── DequeueAt(index) — 특정 슬롯 블록 꺼내기
├── Peek(index) — 꺼내지 않고 조회
└── CreateRandomEntry (Shape 랜덤 + 랜덤 0~3회 회전 적용)

QubeBlockPreviewUI (하단 미리보기 패널 — 드래그 소스)
├── HorizontalLayoutGroup 기반 5슬롯
├── 각 슬롯에 SlotDragHandler (IBeginDragHandler)
├── OnSlotDragStarted 이벤트 → GameManager에 알림
├── 첫 번째 슬롯 1.3x 강조
└── UpdatePreview() — 회전 적용된 형태 미니 셀 렌더링

QubeDifficulty (ScriptableObject — 신규)
├── gridWidth, gridHeight
├── pulseInterval
└── blockShapeCount

QubeScoreFeedback (점수 연출 — 신규)
├── ShowScorePopup(score, position) — "+N" 팝업 (상승+페이드)
├── ShowComboText(count, multiplier) — "x1.5 COMBO!" (스케일+페이드)
└── ShakeScreen(intensity) — Canvas 진동

QubeQuadDetector (영역 감지)
├── FloodFill → 연결 영역 탐색
├── FindQuadsRespectingBlocks → Greedy 직사각형 탐색
└── HighlightQuads → 외곽선/타이머 시각화

QubePulseSystem (턴/소거 관리)
├── pulseInterval (가변, SetPulseInterval() 지원)
├── IncrementTurn → 턴 처리 메인 루프
├── OnPulse(score, removedQuads) — 점수 + Quad 정보 전달
├── ProcessNewQuad → 병합/추가 분기
├── ProcessOverlappingQuads → 겹침 처리 (>= 조건)
├── ProcessAdjacentQuads → 인접 처리 (> 조건)
└── RemoveQuads → 코루틴 소거 (채움 조각도 자동 제거)

QubeQuad (데이터 클래스)
├── cells, bounds, size, turnTimer
├── OverlapsWith / IsAdjacentTo
├── CanMergeWith / MergeWith
└── GetScore (size²×10, 정사각형 1.5x)

QubeCell (개별 셀)
├── 점유 상태/blockId
├── 4방향 Quad 외곽선 (Border, 노란색, Canvas sortingOrder=500)
├── TurnTimer 텍스트 (TMP, Canvas sortingOrder=1000)
└── clearTimer (소거 후 3턴 동안 CLEARED→EMPTY 점진적 보간)
```

### 핵심 게임 루프

```
게임 시작 / 배치 완료 후
  → SpawnNewBlock() — 블록 생성하지 않음, 미리보기 UI만 갱신

프리뷰 슬롯 드래그 시작
  → OnPreviewSlotDragStarted(slotIndex)
    → SpawnBlockFromQueue(slotIndex) — 큐에서 블록 꺼내 생성
    → inputHandler.StartDragFromPreview(block)
  → 드래그 중: 그리드 좌표로 스냅 이동 + Ghost 표시
  → 그리드 위 릴리즈 → Place (배치 가능 시)
  → 그리드 밖 릴리즈 → CancelCurrentBlock() (블록 제거)

배치 시 → PlaceBlockCoroutine()
  → currentBlock.Place()
    → 셀 크기 fullSize→cellSize 축소
    → 같은 블록 인접 셀 사이에 채움 조각(Fill) 생성
  → QubePulseSystem.IncrementTurn()
    → clearTimer 감소 (소거 셀 점진적 색상 복원)
    → 기존 Quad turnTimer++
    → QubeQuadDetector.DetectQuads() (Flood Fill)
    → ProcessNewQuad() (병합/추가)
    → turnTimer >= pulseInterval 인 Quad 소거 (RemoveQuads)
    → HighlightQuads() (시각화 갱신)
  → 미리보기 갱신
  → CanAnyQueuedBlockPlace() 실패 시 GameOver()
```

## 3. 개발 단계 계획

### Phase 0: 블록 회전 제거 & 다음 블록 미리보기 ✅ 완료

**목표**: 플레이어의 블록 회전 조작을 제거하고, 블록 생성 시 랜덤 회전을 적용하여 다양한 형태로 제공. 다음 블록 5개를 미리 결정하여 화면 하단에 표시.

**추가 구현 사항** (Phase 0 과정에서 함께 완료):
- PULSE_INTERVAL 8턴 → 5턴으로 변경
- 1x2 도미노(D) 블록 추가 (블록 5종: L, I, T, O, D)
- 블록 비주얼 구분 개선:
  - 이동 중 블록: fullSize(cellSize+spacing) → 셀 간 구분선 없음
  - 배치 후 블록: cellSize + 같은 블록 인접 셀 사이 채움 조각(Fill) → 하나의 덩어리
  - 다른 블록 간: 자연스러운 그리드 spacing(5px)이 경계선 역할
- Quad 외곽선 렌더링 순서 수정 (Canvas sortingOrder=500으로 PlacedBlocks 위에 표시)
- 소거 셀 점진적 복원: CLEARED_COLOR → EMPTY_COLOR를 3턴에 걸쳐 Lerp 보간 (기존: 검은색 8턴 유지 후 즉시 복원)

#### 0-1. 플레이어 회전 조작 제거 & 생성 시 랜덤 회전 적용

**이유**: 플레이어가 직접 회전하는 대신, 블록 생성 시점에 랜덤 회전(0°/90°/180°/270°)을 적용하여 다양한 형태의 블록을 제공. 이를 통해 조작은 단순화하면서도 블록 다양성은 유지.

**제거 대상 (플레이어 조작)**:

| 파일 | 제거 항목 |
|------|----------|
| `QubeBlock.cs` | `Rotate()` 공개 메서드 (플레이어 호출용) |
| `QubeBlock.cs` | `TryRotateWithWallKick()` 메서드 (Wall Kick 불필요) |
| `QubeBlock.cs` | `WALL_KICK_OFFSETS` 배열 (Wall Kick 불필요) |
| `QubeBlock.cs` | `rotation` 필드 (런타임 추적 불필요) |
| `QubeGameManager.cs:149-157` | Q/E 키 회전 입력 처리 |

**유지/변경 대상 (생성 시 회전)**:

| 파일 | 내용 |
|------|------|
| `QubeBlock.cs` | `CalculateRotatedCells()` → 내부 유틸로 유지 (private static) |
| `QubeBlock.cs` | `ROTATION_COUNT` 상수 유지 |
| `QubeBlockQueue.cs` (신규) | 큐에 블록 추가 시 랜덤 회전 적용 |

**생성 시 회전 메커니즘**:

```csharp
// QubeBlockQueue에서 블록 생성 시
private QubeBlockEntry CreateRandomEntry(QubeBlockShape[] shapes)
{
    QubeBlockShape shape = shapes[Random.Range(0, shapes.Length)];
    int rotationCount = Random.Range(0, 4); // 0, 1, 2, 3회 회전

    // shape.cells를 rotationCount만큼 회전한 셀 배열 생성
    Vector2Int[] rotatedCells = ApplyRotation(shape.cells, rotationCount);

    return new QubeBlockEntry(shape, rotatedCells);
}
```

- O블록(1x1)은 회전해도 동일 → 회전 적용해도 무방 (결과 동일)
- I블록: 0°(가로) / 90°(세로) 2가지 변형
- L블록, T블록: 0°/90°/180°/270° 4가지 변형

**작업 항목**:
1. `QubeBlock.cs`에서 플레이어 회전 관련 코드 제거 (Rotate, TryRotateWithWallKick, WALL_KICK_OFFSETS, rotation 필드)
2. `QubeBlock.cs`의 `CalculateRotatedCells()`는 static 유틸 메서드로 리팩토링
3. `QubeGameManager.cs` HandleInput()에서 Q/E 키 입력 블록 제거
4. `QubeBlockQueue.cs`에서 블록 생성 시 `Random.Range(0, 4)`로 회전 횟수 결정, 셀 좌표에 회전 적용
5. `QubeBlock.Initialize()`에 사전 회전된 셀 배열을 받을 수 있도록 오버로드 또는 파라미터 추가

**영향 범위**:
- Phase 4 모바일 터치 입력에서 "탭 회전", "더블탭 반시계 회전" 항목도 제거 필요
- GDD 대비 미구현 목록에서 모바일 터치 입력 설명 업데이트 필요
- 미리보기 UI에서 회전 적용된 형태 그대로 표시해야 함

#### 0-2. 다음 블록 5개 미리보기 (Next Block Queue)

**핵심 설계**:

```
현재 흐름:
  SpawnNewBlock() → Random.Range()로 즉석 선택 → 1개 생성 (회전 없음)

변경 흐름:
  NewGame() → blockQueue(5개) 미리 생성 (각 블록에 랜덤 회전 적용)
  SpawnNewBlock() → queue에서 Dequeue (사전 회전된 블록) → 새 블록 1개 Enqueue → UI 갱신
```

**데이터 구조**: `QubeBlockEntry` (Shape + 사전 회전된 셀 배열)

```csharp
// 큐에 저장되는 블록 정보 (Shape + 회전 적용된 셀)
public struct QubeBlockEntry
{
    public QubeBlockShape shape;       // 원본 Shape (색상 등 참조)
    public Vector2Int[] rotatedCells;  // 회전 적용된 셀 좌표

    public QubeBlockEntry(QubeBlockShape shape, Vector2Int[] rotatedCells)
    {
        this.shape = shape;
        this.rotatedCells = rotatedCells;
    }
}
```

**새 클래스**: `QubeBlockQueue.cs`

```csharp
public class QubeBlockQueue : MonoBehaviour
{
    private Queue<QubeBlockEntry> queue = new Queue<QubeBlockEntry>();
    private const int QUEUE_SIZE = 5;

    // 큐 초기화 (게임 시작 시) - 5개 랜덤 블록 + 랜덤 회전
    public void Initialize(QubeBlockShape[] shapes);

    // 다음 블록 꺼내기 + 새 블록(랜덤 회전) 추가
    public QubeBlockEntry Dequeue(QubeBlockShape[] shapes);

    // 미리보기용 큐 내용 조회
    public QubeBlockEntry[] GetPreview();

    // 내부: 랜덤 Shape 선택 + 랜덤 회전 적용
    private QubeBlockEntry CreateRandomEntry(QubeBlockShape[] shapes);
}
```

**UI 구성**: 화면 하단에 5개 블록 미리보기 패널

```
┌──────────────────────────────────────┐
│          Qube 게임 그리드             │
│                                      │
│                                      │
│                                      │
├──────────────────────────────────────┤
│  NEXT:  [1] [2] [3] [4] [5]         │
└──────────────────────────────────────┘
```

- 각 미리보기 슬롯은 **회전 적용된** 블록 모양을 축소하여 표시
- 첫 번째(다음 블록)는 약간 더 크게 강조
- 블록 배치 시 큐가 한 칸씩 이동하며 새 블록(랜덤 회전)이 끝에 추가
- 플레이어는 미리보기를 통해 어떤 형태의 블록이 올지 전략적으로 판단 가능

**작업 항목**:
1. `QubeBlockQueue.cs` 신규 생성
   - `Queue<QubeBlockEntry>` 기반 5개 블록 관리 (Shape + 사전 회전 셀)
   - `Initialize()`: 게임 시작 시 5개 랜덤 블록 + 랜덤 회전으로 큐 채움
   - `Dequeue()`: 선두 블록 반환 + 끝에 새 랜덤 블록(랜덤 회전) 추가
   - `GetPreview()`: 현재 큐 내용 배열로 반환
   - `CreateRandomEntry()`: Shape 랜덤 선택 + `Random.Range(0,4)` 회전 적용

2. `QubeBlockPreviewUI.cs` 신규 생성
   - 화면 하단에 HorizontalLayoutGroup 기반 미리보기 패널

## 6. Sprite 기반 비주얼 시스템 (Optional)

**목표**: 현재 단색 `Image`로 렌더링되는 Grid, Cell, Block을 Sprite로 교체할 수 있는 옵션 제공

### 6.1 현재 비주얼 구조

현재 모든 시각 요소가 `UnityEngine.UI.Image`의 단색 `color`로 처리됨:

| 요소 | 파일 | 현재 방식 |
|------|------|----------|
| **Grid 배경** | `QubeGrid.cs` | GridLayoutGroup으로 자동 배치, 별도 배경 없음 |
| **Cell (빈 셀)** | `QubeCell.cs:24` | `Image.color = EMPTY_COLOR (#2D3436)` |
| **Cell (점유)** | `QubeCell.cs:169` | `Image.color = transparent` (블록 비주얼이 위에 덮음) |
| **Cell (소거 흔적)** | `QubeCell.cs` | 3턴 Lerp: `CLEARED_COLOR` → `EMPTY_COLOR` 점진적 보간 |
| **Quad 외곽선** | `QubeCell.cs` | `Image` 4개 (상하좌우), 노란색 — Canvas sortingOrder=500 |
| **Block Cell (이동 중)** | `QubeBlock.cs` | `Image.color = shape.blockColor`, 크기 = cellSize+spacing (구분선 없음) |
| **배치된 Block** | `QubeBlock.cs` | `Image.color = placedColor` (채도 60%), 크기 = cellSize |
| **채움 조각 (Fill)** | `QubeBlock.cs` | 같은 블록 인접 셀 사이 spacing 크기 Image — 블록 덩어리 연결 |

### 6.2 설계 방향: ScriptableObject 기반 스킨 시스템

```csharp
[CreateAssetMenu(menuName = "Qube/Visual Skin")]
public class QubeVisualSkin : ScriptableObject
{
    [Header("Cell")]
    public Sprite cellEmptySprite;       // null이면 기존 단색 사용
    public Sprite cellOccupiedSprite;    // null이면 기존 단색 사용
    public Sprite cellClearedSprite;     // null이면 기존 단색 사용

    [Header("Block")]
    public Sprite blockCellSprite;       // null이면 기존 단색 사용

    [Header("Grid")]
    public Sprite gridBackgroundSprite;  // null이면 배경 없음

    [Header("Border")]
    public Sprite borderSprite;          // null이면 기존 단색 Image

    [Header("Quad Highlight")]
    public Sprite quadHighlightSprite;   // null이면 기존 color * 1.3 방식

    [Header("Fallback")]

}
```

**핵심 원칙**: 모든 Sprite 필드가 `null`이면 현재와 동일하게 단색 렌더링 (하위 호환)

#### Step 1: QubeVisualSkin ScriptableObject 생성

#### Step 2: QubeCell Sprite 지원

**변경 파일**: `QubeCell.cs`

```
현재:

변경:
  if (skin.cellEmptySprite != null)
      image.sprite = skin.cellEmptySprite;
      image.color = skin.useColorTint ? tintColor : Color.white;
  else
      image.sprite = null;  // Unity 기본 흰색 사각형
      image.color = EMPTY_COLOR;
```

**세부 작업**:
1. `QubeCell`에 `QubeVisualSkin` 참조 추가 (Grid에서 주입)
2. `SetColor()` → `ApplyVisual(Color color, CellState state)` 확장
3. `SetOccupied()` 내부에서 상태별 Sprite 교체
4. Border도 Sprite 교체 가능하도록 (`CreateBorderImage` 수정)
5. **기존 동작 유지**: skin이 null이거나 해당 sprite가 null이면 현재 로직 그대로

#### Step 3: QubeBlock Sprite 지원

**변경 파일**: `QubeBlock.cs`

```
현재 (CreateCellVisual):
  Image image = cellObj.AddComponent<Image>();
  image.color = shape.blockColor;

변경:
  Image image = cellObj.AddComponent<Image>();
  if (skin?.blockCellSprite != null)
      image.sprite = skin.blockCellSprite;
      image.color = skin.useColorTint ? shape.blockColor : Color.white;
  else
      image.color = shape.blockColor;
```

**세부 작업**:
1. `QubeBlock.Initialize()`에 skin 파라미터 추가 (또는 Grid에서 참조)
2. `CreateCellVisual()` 수정
3. `Place()` 시 배치된 셀도 동일 Sprite 적용
4. `UpdatePlacementVisualFeedback()` Sprite 모드 대응

#### Step 4: QubeGrid 배경 Sprite

**변경 파일**: `QubeGrid.cs`

**세부 작업**:
1. Grid 뒤에 배경 Image GameObject 추가 (선택적)
2. `skin.gridBackgroundSprite`가 있으면 배경으로 설정
3. 없으면 현재와 동일 (배경 없음)

| `QubeBlock.cs` | `CreateCellVisual`, `Place`, `UpdatePlacementVisualFeedback` | skin == null → 기존 방식 |
| `QubeGrid.cs` | `Awake`에 배경 생성 추가 | sprite == null → 배경 없음 |
| `QubeQuadDetector.cs` | `HighlightQuads` | sprite == null → color * 1.3 |
| `QubeGameManager.cs` | skin 참조 보유, Block 생성 시 전달 | skin == null → 무시 |

**변경 최소화 원칙**:
- 기존 메서드 시그니처 유지 (오버로드 추가)
- null 체크로 분기 → 기존 코드 경로 보존
- 새 ScriptableObject만으로 스킨 교체 (코드 수정 없이 에셋만 변경)

### 6.6 Image.sprite 동작 정리

Unity `Image` 컴포넌트에서:
- `sprite = null` + `color = X` → 단색 사각형 (현재 방식)
- `sprite = S` + `color = White` → Sprite 원본 그대로
- `sprite = S` + `color = X` → Sprite에 tint 적용 (곱연산)

이 특성을 활용하여 `useColorTint` 옵션으로:
- `true`: Sprite + 블록 색상 tint → 블록별 색 구분 유지
- `false`: Sprite 원본 색상만 사용 → 스킨이 색상 완전 제어
