## 2. v2-A: 퍼펙트 클리어 보너스

### 개념

쿼드 소거 결과로 그리드의 **한 행 또는 한 열이 완전히 비워지면** 추가 보너스 점수 + 시각 연출.

### 동작 흐름

```
쿼드 소거 완료
  → 각 행(0~HEIGHT-1) 검사: 모든 셀이 비어있는가?
  → 각 열(0~WIDTH-1) 검사: 모든 셀이 비어있는가?
  → 클리어된 행/열이 있으면:
    → 보너스 점수: 행/열당 500점 × 동시 클리어 수
    → 시각 연출: 클리어된 행/열에 빛 라인 이펙트
    → 사운드: 클리어 효과음
```

### 점수 설계

| 클리어 | 보너스 |
|--------|--------|
| 행 또는 열 1줄 | 500점 |
| 동시 2줄 | 500 × 2 × 1.5 = 1,500점 |
| 동시 3줄+ | 500 × N × 2.0 |
| 전체 클리어 (그리드 완전히 비움) | 5,000점 |

### 변경 파일

| 파일 | 변경 내용 |
|------|----------|
| `QubeGrid.cs` | `CheckClearedLines()` 메서드 추가 — 빈 행/열 목록 반환 |
| `QubePulseSystem.cs` | 소거 후 `CheckClearedLines()` 호출 → 보너스 점수 계산 |
| `QubeGameManager.cs` | 라인 클리어 보너스 점수 반영 + 연출 트리거 |
| `QubeScoreFeedback.cs` | 라인 클리어 연출 추가 (빛 라인 + "PERFECT LINE!" 텍스트) |
| `QubeAudio.cs` | `PlayLineClear()` 효과음 추가 |

### 시각 연출

- 클리어된 행/열 위치에 **가로/세로 빛 라인** (흰색, 0.3초 페이드)
- "PERFECT LINE!" 텍스트 팝업 (금색, 스케일 펀치)
- 전체 클리어 시 "ALL CLEAR!" + 화면 플래시

### 구현 순서

1. `QubeGrid.CheckClearedLines()` 구현
2. `QubePulseSystem.CascadeRemoveChain()` 끝에서 라인 체크 호출
3. 보너스 점수 계산 + `OnPulse` 또는 별도 이벤트로 전달
4. `QubeScoreFeedback`에 라인 클리어 연출 추가
5. `QubeAudio.PlayLineClear()` 효과음 추가

## 3. v2-B: 스타 셀 (목표 셀)

### 개념

게임 시작 시 그리드에 **별 마크 셀 3~5개**를 랜덤 배치. 별 위에 블록을 놓으면 **추가 점수 + 별 수집 연출**. 모든 별을 채우면 새 별 세트가 등장.

### 동작 흐름

```
NewGame() / 별 세트 완료 시
  → 빈 셀 중 랜덤 3~5개에 별 마크 배치
  → 그리드에 별 아이콘 표시 (반투명, 블록 아래 레이어)

블록 배치 시
  → 배치된 셀 중 별이 있는지 확인
  → 별 위에 배치됨:
    → 별 수집 (별당 200점)
    → 별 수집 이펙트 (파티클 + 사운드)
    → 수집 카운터 증가
  → 모든 별 수집 완료:
    → 세트 완료 보너스 (1,000점)
    → 새 별 세트 배치 (빈 셀에)
```

### 데이터 설계

```csharp
// QubeGrid에 추가
private HashSet<Vector2Int> starCells;      // 현재 별 위치
private int starSetSize = 3;                // 세트당 별 개수 (점진적 증가 가능)

public void PlaceStars(int count);          // 랜덤 빈 셀에 별 배치
public bool HasStar(Vector2Int pos);        // 별 존재 여부
public void CollectStar(Vector2Int pos);    // 별 수집
public bool AllStarsCollected();            // 모든 별 수집 여부
```

### 변경 파일

| 파일 | 변경 내용 |
|------|----------|
| `QubeGrid.cs` | 별 데이터 관리 (`starCells`) + 별 비주얼 (Image 레이어) |
| `QubeBlock.cs` | `Place()` 에서 별 수집 체크 |
| `QubeGameManager.cs` | 별 초기화, 수집 점수, 세트 완료 처리 |
| `QubeScoreFeedback.cs` | 별 수집 연출 ("★ +200" 팝업), 세트 완료 연출 |
| `QubeAudio.cs` | `PlayStarCollect()` 효과음 추가 |

### 시각 디자인

- **별 마크**: 금색 반투명 별 아이콘 (프로시저럴 텍스처 또는 TMP "★")
- **위치**: cellBackgrounds 위, PlacedBlocks 아래 레이어
- **수집 시**: 별이 위로 떠오르며 점수 텍스트로 흡수되는 애니메이션
- **새 세트 등장**: 별이 하나씩 톡톡 나타나는 연출 (0.1초 간격)

### 구현 순서

1. `QubeGrid`에 별 데이터 + 비주얼 관리 추가
2. `QubeBlock.Place()`에서 별 수집 로직 연동
3. `QubeGameManager`에서 별 초기화 + 세트 완료 처리
4. 별 수집 연출 + 효과음
5. UI에 별 수집 카운터 표시 (선택)

## 4. v2-C: 연쇄 소거 (도미노 효과)

### 개념

쿼드 소거 후 셀이 제거되면서 **남은 블록들로 새로운 쿼드가 형성**될 수 있음. 이를 자동으로 감지하고 **연쇄 소거** 발동. 연쇄마다 배율 증가 + 연출 강화.

### 동작 흐름

```
쿼드 소거 (기존 흐름)
  → 셀 제거 + 캐스케이드 애니메이션
  → 소거 완료 후 0.3초 대기
  → QubeQuadDetector.DetectQuads() 재실행
  → 새 쿼드 발견?
    → Yes: 연쇄 소거 발동 (연쇄 깊이 +1)
      → 배율 증가 (1x → 1.5x → 2x → 3x)
      → "2 CHAIN!" 텍스트 팝업
      → 화면 셰이크 강도 증가
      → 다시 소거 → 재감지 (반복)
    → No: 연쇄 종료
      → 연쇄 2회 이상이면 "TOTAL: +N,NNN" 표시
```

### 연쇄 배율

| 연쇄 깊이 | 배율 | 연출 |
|-----------|------|------|
| 1 (첫 소거) | 1.0x | 기본 |
| 2 | 1.5x | "2 CHAIN!" + 셰이크 |
| 3 | 2.0x | "3 CHAIN!" + 강한 셰이크 |
| 4+ | 3.0x | "AMAZING!" + 화면 플래시 |

### 현재 코드와의 차이

현재 `QubePulseSystem`은 쿼드 클릭으로 수동 소거만 지원. 연쇄 소거는:

1. **소거 후 자동 재감지** — 현재는 `ProcessTurn()`에서만 감지
2. **연쇄 깊이 추적** — 현재 없음
3. **연쇄 종료 시 총점 표시** — 현재 없음

### 변경 파일

| 파일 | 변경 내용 |
|------|----------|
| `QubePulseSystem.cs` | `CascadeRemoveChain()` 끝에서 재감지 → 연쇄 루프, 연쇄 깊이 추적 |
| `QubeQuadDetector.cs` | `DetectQuads()` — 기존 로직 재활용 (변경 없음) |
| `QubeGameManager.cs` | 연쇄 점수 누적, 연쇄 종료 시 TOTAL 표시 |
| `QubeScoreFeedback.cs` | "N CHAIN!" 텍스트, TOTAL 팝업, 연쇄 깊이별 셰이크 강도 |
| `QubeAudio.cs` | `PlayChain(int depth)` — 연쇄 깊이에 따라 피치 상승하는 효과음 |

### 핵심 구현: 연쇄 코루틴

```
IEnumerator ChainRemoveCoroutine(List<QubeQuad> initialQuads)
{
    int chainDepth = 0;
    List<QubeQuad> currentQuads = initialQuads;

    while (currentQuads.Count > 0)
    {
        chainDepth++;
        float multiplier = GetChainMultiplier(chainDepth);

        // 현재 쿼드들 소거 (캐스케이드 애니메이션)
        yield return CascadeRemoveChain(currentQuads);

        // 점수 계산 + 이벤트
        int score = CalculateScore(currentQuads) × multiplier;
        OnChainPulse?.Invoke(score, chainDepth, currentQuads);

        // 대기 후 재감지
        yield return new WaitForSeconds(0.3f);
        currentQuads = quadDetector.DetectQuads();
        // 새로 감지된 쿼드가 있으면 연쇄 계속
    }

    if (chainDepth >= 2)
        OnChainComplete?.Invoke(totalScore, chainDepth);
}
```

### 구현 순서

1. `QubePulseSystem`에 연쇄 루프 코루틴 추가
2. 연쇄 깊이별 배율 + 이벤트 시스템
3. `QubeScoreFeedback`에 "N CHAIN!" + TOTAL 연출
4. `QubeAudio.PlayChain()` 효과음 (깊이별 피치 상승)
5. 연쇄 중 입력 차단 (Phase 유지)

## 5. 구현 우선순위 & 의존성

```
v2-A (퍼펙트 클리어)  ← 독립, 가장 쉬움
v2-B (스타 셀)        ← 독립, 쉬움
v2-C (연쇄 소거)      ← v2-A 이후 권장 (라인 클리어 + 연쇄가 조합되면 시너지)
```

### 추천 구현 순서

| 순서 | 기능 | 예상 작업량 | 이유 |
|------|------|-----------|------|
| 1 | **v2-A 퍼펙트 클리어** | 작음 | 기존 코드에 검사 로직만 추가 |
| 2 | **v2-B 스타 셀** | 중간 | 독립적, 즉각적 재미 증가 |
| 3 | **v2-C 연쇄 소거** | 큼 | PulseSystem 코루틴 재설계 필요 |
