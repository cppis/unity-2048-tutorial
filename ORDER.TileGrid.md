# Tile Grid 셀 좌표 주의사항  

## 문제 상황

일반적인 좌표계와 Unity UI 좌표계의 차이 때문입니다:  

일반적인 수학/게임 좌표계:  
      ↑ y (위로 갈수록 증가)  
      |  
      | 
      ───→ x  

  Unity UI 좌표계:  
  ───→ x  
  |  
  |  
  ↓ y (아래로 갈수록 증가)  

  Unity UI는 **왼쪽 상단이 (0,0)**이고, 아래로 갈수록 y값이 증가합니다.  

## 구체적인 예시  

4x4 그리드에서 좌표를 생각해보면:

Row 0: [0,0] [1,0] [2,0] [3,0]  ← y=0 (맨 위)  
Row 1: [0,1] [1,1] [2,1] [3,1]  ← y=1  
Row 2: [0,2] [1,2] [2,2] [3,2]  ← y=2  
Row 3: [0,3] [1,3] [2,3] [3,3]  ← y=3 (맨 아래)  

이제 [1,1] 위치에서 위로 이동한다고 가정하면:  
  - 방향 벡터: direction = (0, 1) (위쪽 = +y 방향)  
  - 목표 셀: [1,0] (y 좌표가 감소해야 함)  

## TileGrid.cs:40의 해결책  

coordinates.x += direction.x;  // x는 그대로 더함  
coordinates.y -= direction.y;  // y는 빼기로 반전!  

계산 과정:
  - 현재: [1,1]
  - 방향: (0, 1) (위쪽)
  - 결과: x = 1 + 0 = 1, y = 1 - 1 = 0
  - 최종: [1,0] ✅ 올바르게 위쪽 셀!

만약 -=가 아니라 +=를 사용했다면:  
  - 결과: [1,2] ❌ 아래쪽 셀로 잘못 이동!  

## 요약  
  TileGrid.cs:40에서 y -= direction.y를 사용하는 이유는 Unity UI의 y축이 아래로   증가하지만, 게임 로직에서는 위쪽을 +y로 생각하기 때문에 반전이 필요합니다.
