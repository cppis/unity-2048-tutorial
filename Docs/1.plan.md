## 2. 아키텍처 현황

### 클래스 구조

```
QubeGameManager (싱글톤, DefaultExecutionOrder -1)
├── 프리뷰 슬롯 드래그 이벤트 구독 (OnPreviewSlotDragStarted)
├── EnterDraggingBlock() — 큐에서 블록 생성 + 드래그 시작
├── CancelCurrentBlock() — 드래그 취소 시 블록 제거 + 큐 복원
├── CanAnyQueuedBlockPlace() — 게임오버 판정 (큐 전체 검사)
├── 난이도 시스템 (QubeDifficulty ScriptableObject)
├── 프리뷰 회전: RotatePreview 액션 → blockQueue.RotateFirst() + 애니메이션
├── QubeAudio 초기화 + PreviewUI 연결
├── 점수/UI 업데이트
└── 게임 상태 (isGameOver, GamePhase 4단계)

QubeInputHandler (입력 통합)
├── 터치: 그리드 위 드래그 + 릴리즈 배치
├── 프리뷰 드래그: StartDragFromPreview() → 그리드 스냅 이동
├── 드래그 취소: 그리드 밖 릴리즈 → CancelDrag 액션
├── 스와이프 회전: 하단 40% 좌우 스와이프 (80px 임계값) → RotatePreview 액션
├── 연속 회전: 스와이프 유지 시 80px마다 추가 회전
└── ScreenToGridPosition() — 화면 좌표 → 그리드 좌표 변환

QubeGrid (가변 크기 그리드 컨테이너)
├── static WIDTH/HEIGHT (SetSize()로 변경 가능)
├── cells[WIDTH, HEIGHT] 배열 (동적 할당)
├── cellBackgrounds[,] — 셀 배경 Image 배열 (소거 비주얼용)
├── PlacedBlocks 컨테이너 (배치된 셀 + 채움 조각)
├── IsGridEmpty() — 빈 그리드 여부 확인
├── HasAdjacentOccupied() — 인접 점유 셀 확인
├── UpdateCellVisuals() — QubeCell.cellColor를 배경 Image에 반영
└── RemoveFillPieces() — 소거 시 채움 조각도 함께 제거

QubeBlock (드래그 조작 블록)
├── QubeBlockShape (ScriptableObject)
├── 이동: Move() + SetPosition() (그리드 스냅) + SetScreenPosition() (그리드 밖)
├── 드래그 고스트: 반투명 채움(alpha 0.15) + 아웃라인 보더(4.5px, alpha 0.7)
├── 배치 시: EndDragVisual()로 불투명 복원 → Place()
├── Ghost 미리보기 (아웃라인만, 배치 가능: 흰색, 불가: 빨간색)
│   ├── 투명 내부 (그리드 라인 그대로 보임)
│   ├── Canvas overrideSorting (sortingOrder=100, PlacedBlocks 위)
│   └── 고스트 표시 시 블록 셀 숨김 (아웃라인만 표시)
├── 그리드 밖 드래그 시 셀 숨김
├── 인접 배치 규칙: 그리드 비어있으면 자유, 아니면 기존 블록에 인접해야 함
├── Place() — 셀 활성화 복원 + 같은 블록 인접 셀 사이에 채움 조각(Fill) 생성
├── Initialize() 오버로드 — 사전 회전된 셀 배열 수신
└── ApplyRotation() static — 큐에서 사전/수동 회전 시 사용

QubeBlockQueue (다음 블록 큐)
├── Queue<QubeBlockEntry> (5개, Shape + 사전 회전 셀)
├── Initialize(shapes, useShapeCount=-1) — null 요소 필터링 후 사용
├── Dequeue / GetPreview
├── RotateFirst(direction) — 첫 번째(활성) 블록만 ±90° 회전
├── DequeueAt(index) — 특정 슬롯 블록 꺼내기
├── PushFront(entry) — 취소 시 큐 맨 앞에 복원
└── CreateRandomEntry (Shape 랜덤 + 랜덤 0~3회 회전 적용)

QubeBlockPreviewUI (하단 미리보기 패널 — 드래그 소스)
├── HorizontalLayoutGroup 기반 5슬롯
├── 첫 번째 슬롯: 밝은 배경(0.18,0.22,0.28) + 시안 글로우 보더 + 1.3x 스케일
├── OnSlotDragStarted 이벤트 → GameManager에 알림
├── AnimateRotateFirst() — 블록 셀만 회전 (배경 고정), EaseOutBack 0.2초
├── 빠른 연속 회전 시 즉시 스냅 (애니메이션 생략)
├── 틱 사운드 재생 (회전 시작 시 1회)
└── UpdatePreview() — 회전 적용된 형태 미니 셀 렌더링 (베벨 없음)

QubeBlockShape (ScriptableObject — 블록 모양 정의)
├── 14종 고유 모양 (회전 중복 4쌍 제거)
├── 18색 팔레트 (14종에 순서대로 적용)
└── ApplyPalette() — null 안전, blockShapes 배열에 색상 자동 적용

QubeBevel (유틸리티 — 현재 미사용)
└── AddBevel() — 하이라이트/섀도우 바 추가 (Phase 3에서 모든 호출 제거)

QubeAudio (프로시저럴 사운드)
├── Initialize() — AudioSource + AudioClip 생성
├── PlayTick() — 회전 틱 (15ms, 3200Hz 사인파 + 노이즈)
├── PlaySnap() — 배치 스냅 (150ms, 저음 금속 흡착)
└── AudioClip.Create로 런타임 생성 (외부 에셋 불필요)

QubeDifficulty (ScriptableObject)
├── gridWidth, gridHeight
├── pulseInterval
└── blockShapeCount (최대 10)

QubeAdBanner (AppLovin MAX 배너 광고)
├── Initialize() — SDK 초기화 + 배너 생성
└── #if !UNITY_EDITOR 분기

QubeScoreFeedback (점수 연출)
├── ShowScorePopup(score, position) — "+N" 팝업 (상승+페이드)
├── ShowComboText(count, multiplier) — "x1.5 COMBO!" (스케일+페이드)
└── ShakeScreen(intensity) — Canvas 진동

QubeQuadDetector (영역 감지)
├── FloodFill → 연결 영역 탐색
├── FindQuadsRespectingBlocks → Greedy 직사각형 탐색 (최소 6셀, 2x2 초과)
└── HighlightQuads → 외곽선/타이머 시각화

QubePulseSystem (턴/소거 관리)
├── pulseInterval (가변, SetPulseInterval() 지원)
├── ProcessTurn → 턴 처리 메인 루프
├── OnPulse(score, removedQuads) — 점수 + Quad 정보 전달
└── RemoveQuads → 코루틴 소거

QubeBackground (프로시저럴 배경)
├── 수직 그라데이션 + 비네트 (Texture2D 생성)
└── 앰비언트 파티클 (10개 반투명 빛 입자)

QubeUIParticles (UI 파티클)
├── EmitPlaceParticles — 블록 배치 시 방사형 버스트
└── EmitQuadParticles — 쿼드 완성 시 폭발
```

### 핵심 게임 루프

```
Phase 1: QubeControl (대기)
  → 프리뷰 슬롯 터치 → OnPreviewSlotDragStarted → Phase 2
  → 하단 좌우 스와이프 → 첫 번째 블록 ±90° 회전 + 프리뷰 애니메이션 + 틱 사운드
  → Quad 클릭 → 소거

Phase 2: DraggingBlock (드래그)
  → 블록이 고스트(반투명+아웃라인)로 표시
  → 그리드 위: 그리드 좌표로 스냅, Ghost 아웃라인 표시 (블록 셀 숨김)
    → 배치 가능: 흰색 아웃라인
    → 배치 불가: 빨간색 아웃라인
  → 그리드 밖: 셀 + 고스트 숨김
  → 그리드 위 릴리즈 → Phase 3 (배치 시도)
  → 그리드 밖 릴리즈 → 즉시 취소, 큐 복원 → Phase 1

Phase 3: PlacingBlock (배치)
  → EndDragVisual() — 고스트 → 실제 블록 복원
  → currentBlock.Place() — 셀 활성화 + 배치 + 채움 조각 생성
  → 인접 배치 규칙 검증 (빈 그리드 제외)
  → 배치 파티클 + 바운스 애니메이션
  → Phase 4

Phase 4: UpdatingQube (업데이트)
  → QubePulseSystem.ProcessTurn()
  → Quad 감지/하이라이트
  → CanAnyQueuedBlockPlace() 실패 시 GameOver()
  → Phase 1 복귀
```

### Phase 1: 플레이 편의성 개선 — 완료

- Ghost 배치 미리보기
- 점수/콤보 피드백 (팝업, 콤보, 셰이크)
- 난이도 시스템 (QubeDifficulty ScriptableObject)
- 터치/드래그 입력 통합
- 프리뷰 슬롯 드래그 배치
- 드래그 취소 시 큐 복원

### Phase 2: 블록 회전 복원 & 확장 — 완료

- **프리뷰 블록 회전**: 하단 영역 좌우 스와이프로 첫 번째 블록 ±90° 회전
- **프로시저럴 사운드**: QubeAudio.cs — 회전 틱 + 배치 스냅
- **프리뷰 첫 슬롯 강조**: 밝은 배경색
- **드래그 고스트 비주얼**: 반투명 채움 + 아웃라인 보더
- **그리드 밖 고스트 숨김 / 릴리즈 즉시 취소**
- **인접 배치 규칙**: 빈 그리드 제외, 기존 블록에 인접해야 배치 가능
- **배치 불가 시 쿼드 자동 폭파**
- **게임오버 판정 + 연출**
- **블록 16종 + 16색 팔레트**

### Phase 3: 비주얼 정리 & 블록 중복 제거 — 완료

- **베벨 제거**: QubeBevel.AddBevel() 호출 전부 제거 (QubeBlock, QubeBlockPreviewUI)
- **블록 셀 크기 변경**: `cellSize + spacing` → `cellSize` (그리드 라인 노출)
- **Ghost 아웃라인 개선**:
  - 색상: 흰색 (배치 가능) / 빨간색 (배치 불가)
  - 두께: 6.75 → 10.125 (1.5배)
  - 내부 완전 투명 (그리드 라인 그대로 보임)
  - Canvas overrideSorting (sortingOrder=100) — PlacedBlocks 위, QuadOutlines(500) 아래
  - 고스트 표시 시 블록 셀 숨김 (아웃라인만)
  - 그리드 범위 내이면 항상 표시 (배치 불가 시에도)
- **블록 중복 제거**: 회전 시 같은 모양이 되는 4쌍 제거 (18종 → 14종)
  - 제거: 04(J), 13(W), 14(Wm), 15(Y)
- **null 안전성**: ApplyPalette, QubeBlockQueue.Initialize에서 null 요소 처리

## 8. GDD vs 구현 비교표

| GDD 항목 | GDD 내용 | 현재 구현 | 차이 |
|----------|---------|----------|------|
| 그리드 | 8x6 | 가변 (난이도별: 10x8 / 8x6) | 난이도 시스템으로 해결 |
| 블록 | L,I,T,O + S,Z(Chain Hard) | **14종** (회전 중복 제거) | 중복 제거, 거울상 유지 |
| 블록 회전 | 플레이어 조작 | **프리뷰 스와이프 회전** (생성 시 랜덤 + 수동 회전) | 의도적 변경 |
| 블록 배치 | 그리드 위 직접 조작 | **프리뷰 드래그 → 고스트 → 배치** | 의도적 변경 |
| 배치 규칙 | GDD 미정의 | **인접 배치 필수** (빈 그리드 제외) | 신규 규칙 |
| 배치 미리보기 | GDD 미정의 | Ghost 아웃라인 (흰색/빨간색) | 신규 기능 |
| 다음 블록 미리보기 | GDD 미정의 | 5개 큐 + 하단 드래그 UI + 회전 | 신규 기능 |
| 블록 비주얼 | GDD 미정의 | 플랫 (베벨 없음), cellSize 크기 | Phase 3 변경 |
| Zen 펄스 | 4턴마다 전체 소거 | 가변 (난이도별: 4~7턴) | 난이도 시스템으로 해결 |
| Chain 모드 | Quad별 타이머 + 연쇄 | 미구현 | 핵심 기능 부재 |
| 점수 피드백 | 팝업, 셰이크 | **팝업 + 콤보 + 셰이크** | 구현 완료 |
| 입력 | 터치 (드래그, 탭) | **터치/드래그 + 스와이프 회전** | 구현 완료 |
| 난이도 | GDD 미정의 | **Easy/Normal/Hard ScriptableObject** | 신규 기능 |
| 사운드 | 배치/소거/연쇄 효과음 | **회전 틱 + 배치 스냅** (프로시저럴) | 소거/연쇄 미구현 |
| 게임오버 | Zen 없음, Chain 있음 | 큐 전체 블록 배치 불가 시 게임오버 | Zen 불일치 |
